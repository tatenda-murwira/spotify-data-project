# Spotify Listening History Analysis Project

A personal data analysis project that processes Spotify Extended Streaming History data (2022–2026) to uncover listening patterns, behavioral insights, and music trends.

---

## Project Structure

```
Spotify Extended Streaming History/
├── Spotify data project.ipynb   # Main analysis notebook
├── spotify_dashboard.py         # Interactive Streamlit dashboard
├── llm_chat.py                  # AI-powered chat interface (Google Gemini)
├── clean_spotify_history.csv    # Cleaned & exported data (generated by notebook)
├── Streaming_History_Audio_*.json  # Raw Spotify data files (2022–2026)
└── Streaming_History_Video_2024.json
```

---

## What the Notebook Does

The notebook is a step-by-step ETL + analysis pipeline:

| Cell | Purpose |
|------|---------|
| **Cell 1** | Installs `sqlalchemy` and `psycopg2-binary` for optional PostgreSQL export |
| **Cell 2** | **ETL Pipeline** — Loads all JSON files, converts timestamps to local timezone (Africa/Harare), calculates `minutes_played`, filters skipped tracks (<30s), renames columns, and drops nulls (podcasts) |
| **Cell 3** | **Export** — Saves the cleaned DataFrame to `clean_spotify_history.csv` |
| **Cell 4** | **Deep Work Sessions** — Detects listening sessions >2 hours with gaps >15 min; plots when (hour of day) deep work sessions occur |
| **Cell 5** | **Anchor Artists & Discovery Rate** — Finds artists in your Top 20 in both 2022 and 2026 ("musical anchors"), and counts unique artists per year to measure music discovery |
| **Cell 6** | **Deep Work (Refined)** — Re-runs session detection with a zero-skip filter to find truly focused listening sessions |
| **Cell 7** | **Exam/Study Spikes** — Plots monthly listening volume to visually identify high-intensity study periods |
| **Cell 8** | **Skip Rate YoY** — Calculates year-over-year skip rate (% of tracks played <30s) as a proxy for music satisfaction |
| **Cell 9** | **Musical Anchors (Refactored)** — Cleaner function-based version of the anchor artist analysis |
| **Cell 10** | **Discovery Season** — Groups unique artist count by month to identify exploration vs. comfort-zone phases |
| **Cell 11** | **Top 10 Artists Diagnostic** — Lists your top 10 most-played artists (used to pick an artist for burnout analysis) |
| **Cell 12** | **Artist Burnout Curve** — Tracks daily listening volume for a chosen artist, marks the peak obsession date, and detects when listening dropped 80% (burnout point) |
| **Cell 13** | **Homegrown vs. International** — Classifies artists as local (Zimbabwean) or international and shows the percentage split per year |

## The Dashboard (`spotify_dashboard.py`)

An interactive Streamlit web app with 5 tabs:

- **Overview** — Listening by day of week, hour of day, and a full heatmap
- **Trends** — Monthly listening trend, yearly comparison, skip rate over time
- **Top Artists & Tracks** — Ranked lists and bar charts of your top 10
- **Search** — Regex-powered search for artists and tracks
- **2026 Wrapped** — A Spotify Wrapped-style summary for Jan–Apr 2026 with comparisons to 2025

### Running the Dashboard
```bash
streamlit run spotify_dashboard.py
```

## The AI Chat (`llm_chat.py`)

A Google Gemini-powered chat interface that lets you ask natural language questions about your listening data. It generates and executes Python/pandas code on the fly to answer queries like:

- *"What's my most played song in 2024?"*
- *"How many hours did I listen to Coldplay?"*
- *"Which month had the highest skip rate?"*

### Usage (called from another script or Streamlit page)
```python
from llm_chat import chat
answer = chat("Who is my top artist?", df, history=[], api_key="YOUR_GEMINI_KEY")
```

## Key Concepts & Definitions

| Term | Definition |
|------|-----------|
| **Skip** | A track played for less than 30,000ms (30 seconds) |
| **Session** | A continuous block of listening with gaps no longer than 15 minutes |
| **Deep Work Session** | A listening session lasting 120+ minutes (used as a proxy for focused work/study) |
| **Anchor Artist** | An artist who appears in your Top 20 in both the first and last year of the dataset |
| **Burnout Point** | The first day after peak listening where daily volume drops to ≤20% of the peak |
| **Timezone** | All timestamps are converted to **Africa/Harare (UTC+2)** |
